## 0. 対象の確定（design-reread Step 0 ゲートの結果）

- 設計対象＝**概算量産見積**。仕様・素材が固まる前に「価格・MOQ・条件」を客へ出して交渉テーブルに乗せ、**発行履歴として保存する**（いつ・いくら・どの程度のミニマムで・どの条件で出したか）。
- 排他確認: qe-0 §3 末尾「用尺・取り切りは量産(QE-1)専用」は*計算方式*の排除であって、概算（BOM 連動なし・経験値/過去実額ベース）とは層が違い抵触しない。「概算」は qe-0 に 0 件＝新概念。
- 軸の整理（旧 (あ)(い) ラベルは廃止）: サンプル軸と量産軸は別物。量産軸の中をさらに「概算レーン（本書）」と「確定計算レーン（QE-1 v1.0）」に分ける。**サンプルの概算見積は不要**（標準形）。

## 1. 位置づけ（独立レーン・提示価格を出す）

- QE-1R は量産軸の**概算レーン**＝確定前・経験値/過去実額・保存型の見積発行。客に見せる数字は**提示価格**（原価ではなく利益込み）。
- QE-1（確定計算レーン）との関係: **時間軸でのみ接続**する（概算で交渉 → 仕様確定 → 確定計算 QE-1）。計算層の「前段」ではなく別系統。QE-1 の自己定義は「量産見積の*計算*・原価まで・売価/マージン含まない」であり、概算（客に出す価格・MOQ・条件・保存）とは性質が違うため、同一レーンに置かない。
- staging への座席: qe-0 §1 の QE-0/1/2/3+ に概算の席は無かった。QE-1R を**独立の席**として後付けする。

## 3. 保存箱＝軽量新設（論点1・確定）

**結論: 概算専用の軽量モデルを新設する（仮称 `RoughEstimate` ＋ 明細 `RoughEstimateItem`）。既存 Quotation には相乗りしない。**

現物根拠（2026-07-01 recon）:

- Quotation は**確定見積書(QE-2)前提の重い箱**。必須列に `quotationNumber`（採番・@@unique）・`versionNumber`・`clientId`・`validUntil`（有効期限・default 無し）。さらに提出/承認/ロック/緊急進行/却下/受注変換のライフサイクル一式を持つ。概算＝交渉前の使い捨て発行履歴とは粒度が二段違う。
- `QuotationType` に量産概算を表す値が無い（SAMPLE / MASS / FINAL / CUSTOM）。流用すると MASS+DRAFT か CUSTOM で「概算らしきもの」を曖昧運用することになり、QE-2 実装時に概算と確定見積が同一テーブルに混在する。
- MOQ: `Quotation.hasMoqTiers` ＋ `QuotationMoqTier` は確定見積の階段単価用。概算の提示 MOQ をそこに入れると QE-3+ の階段計算と混線する。
- Product↔Quotation の戻り線は無い（`productId` は scalar FK）。流用しても「配線が既存」という利点は出ない。

帰結: 流用の利点（配線既存）は無く、欠点（必須列の無理な充足・型値の曖昧運用・将来の混在）だけが残る。よって新設が素直。

**重要（実装重量）**: 新設は **migration＝本番スキーマ変更を伴う**。QE-1 v1.0（計算ビューのみ・スキーマ変更なし）と異なり、概算は保存型ゆえスキーマが要る。実装は triple-gate（dev 確認 → 本番 dry-run ROLLBACK＋件数確認 → 本番 commit）対象。

将来の語彙統一: 後で確定見積へ引き継ぐ可能性に備え、概算明細の費目・通貨の命名は確定側（`costCategoryId` / `currency`）と揃える。

## 4. source（過去実額の引き当て）

- 一次 source: 過去 `PoItem`（materialId キーで `unitPrice` 検索）＋ 過去 `WoItem`（costCategoryId キー）。いずれも `unitPrice`(nullable)・`currency` 列が実在（recon 済み）。「この生地は前回いくら」を materialId で呼び出す。
- 引き当て方式: **参照リンクではなくスナップショットコピー**。引き当て時点の値を概算明細に焼き込み、後で元 PO/WO が変わっても概算は動かさない（概算は「その時点でこう出した」発行履歴だから）。
- 二次 source 候補: 過去の確定見積（Quotation）も将来は引き当て元になり得るが、Quotation 生成ロジックが休眠のため v0.1 では PoItem/WoItem のみ。

## 5. 価格の作り方（自動計算 → 手打ちで整える）★本書の中核

概算の数字は「機械が既定値を入れ、人が最後に上書きして整える」という一本の作法で貫く。

### 5-1. 自動計算レイヤー（下敷き・根拠）
- **原価**（材料費＋工賃）＝ §4 の過去実額引き当て（スナップショットコピー）＋手入力を自動集計。
- **提示価格（自動）** ＝ 原価 ×(1＋利益率)。
  - 利益率は**デフォルト値が入っているが変更可能**。粒度は**概算1件に1つ**（費目別の率は QE-3+ 領分＝使わない）。
  - デフォルト利益率の出どころ＝ **`Brand.defaultMarginRate`**（品番 → ブランド → 既定率。既存列を使用・migration 追加なし）。
- **初期費用**（版代・パンチ代等）＝ 各項目に**デフォルト値が入っているが変更可能**。初期費用も **原価 ×(1＋利益率)** で価格化する（利益率は上と同じ・デフォルト入り・変更可）。ただし計上は別枠を崩さない（§6）。
- **利益率の適用範囲**: 利益率は**材料費・工賃・初期費用のすべてに掛ける**。初期費用も価格化の対象。

### 5-2. 手打ちレイヤー（最終確定）
- 最終的な提示見積金額・（将来の）納品金額は**人が手打ちで整える**。
- 手打ち入力欄には **5-1 の自動計算金額がデフォルトで入っている**。人がその上に上書きして仕上げる。

### 5-3. 保存の作法（★列設計に直結）
- **自動計算値**（原価・適用利益率・自動提示価格・自動初期費用価格）と、**手打ちの最終値**（人が整えた見積/納品金額）を、**別列で両方保存する**。
- 手打ちで自動値を潰さない。自動値＝「なぜその価格にしたか」を辿る根拠（再現性）、手打ち最終値＝「実際に客へ出した数字」。発行履歴の「いくら」は手打ち最終値が正。

## 6. 絶対防衛線（初期費用の別枠計上）

- 初期費用（パターン代・版代・型代・刺繍パンチ代・グレーディング代・サンプル代）は別枠「**別途請求項目（1枚原価外）**」に項目別表示。**1枚原価の分子に合算しない**。
- product-sample §6-2 の確定: 個別売り立て（`INDIVIDUAL_BILLING`）＝上記初期費用群／製品単価インクルード＝加工工賃そのもの（洗い・プリント等）。
- **レイヤーの切り分け（重要）**: 「初期費用に利益を乗せる（価格）」と「初期費用を1枚原価に混ぜない（原価集計）」は別レイヤーで、両立する。
  - 価格の話: 初期費用も 原価×(1＋利益率) で価格化する（§5-1）。
  - 原価集計の話: 初期費用は別枠のまま。1枚原価の分子には入れない。**この防衛線は崩さない**。
- 慎太郎さんの線引き: (1)加工費 (2)材料費 が多少混ざるのは許容／**(3)初期費用が1枚原価に混ざるのは絶対ダメ**（別枠計上を維持）。
- ★将来フェーズのバックログ（防衛線の例外オプション）: 稀に客要望で「初期費用を量産（1枚単価）へインクルーズしてほしい」ケースがある。その**切替設定を先のフェーズで用意する**（v0.1 スコープ外）。これは防衛線を破るのではなく、客要望時に限って選べる明示的な切替として実装する。デフォルトは常に別枠計上。

## 7. 通貨

- 行ごとに JPY/USD。合計は JPY 換算で提示。基本通貨は**ドル円のみ**・CNY/VND/EUR は対象外（silent fallback 禁止）。
- 換算レートの持ち方（入力レートか参照か）は実装ブリーフ段。QE-1 v1.0 は入力レートで JPY 換算する方針なので、概算もそれに揃える見込み。

## 8. 論点 2〜5 の方針（確定）

**論点2 — 保存項目の「条件」の中身**
- 確定: 品番・発行日・（提示）価格・MOQ。
- 「条件」＝ (a) 通貨前提 (b) 想定数量帯 (c) 前提メモ（素材グレード仮定・色数・納期前提等を自由記述）。
- 概算は使い捨てゆえ Quotation のような `validUntil` 必須にはしない。有効期限は**任意の参考列**に留める。

**論点3 — 手入力 vs 過去引き当ての混在**
- 1つの概算内で**混在 OK**。明細行ごとに source（`MANUAL` / `PAST_PO` / `PAST_WO`）を enum で記録する。BomItem の `costSource`/`usageSource` と同じ house style。

**論点4 — MOQ の持ち方**
- 概算ヘッダに**単純な提示 MOQ 値1つ**。階段（QuotationMoqTier 相当）は持たない＝QE-3+ 送り。

**論点5 — 永続化と確定見積への引き継ぎ動線**
- v0.1 は**保存＋出力（表示）まで**。確定見積（QE-1/QE-2）への引き継ぎ動線は含めない（引き継ぎ先が休眠のため、引き継ぎは先方実装とセットで初めて価値が出る）。将来に備え語彙だけ揃える（§3 末尾）。

## 9. スコープ外（非引き込み）

- サンプルの概算見積（不要・標準形）。
- 品番からの候補提案 → B-038。
- 粗 BOM 計算（重い・概算では不採用）。
- **マージン4階層・費目別利益率・厳密な売価計算 → QE-3+**。※ 概算は「1件1率のざっくり利益率（全費目一律）」までがスコープ内（§5-1）。それ以上の階層化はしない。
- **初期費用を1枚単価へインクルーズする切替設定 → 将来フェーズ**（§6 バックログ）。v0.1 は常に別枠計上。
- QE-1 の取り切り計算（量産確定レーン）。
- QE-1 の「ROLL/METER vs 指定数 2本立て未反映」問題は量産(QE-1)側の宿題。**概算には引き込まない**（スコープ逆流防止）。
