## 1. 概要

shunya 生産管理システムの**全マスターに対する CSV インポート/エクスポート機能**を、
Phase 1A 完了後に一括実装する。

### 着手タイミング

- Phase 1A-13（生地・副資材）完了後
- すべてのマスターが CRUD ベースで動作する状態になってから

### 着手理由

- 仕様書（`docs/20260516_10_other_final.prisma`）で `CsvImportJob` / `DataExportJob` が**全マスター共通**として設計
- 各マスターで CSV を個別実装すると DRY 違反
- 全マスター揃ってから一気に作る方が、汎用機能として高品質に作れる

## 3. 機能要件

### 3.1 CSV エクスポート（2 用途）

#### 用途 ①: テンプレート（空のヘッダーのみ）ダウンロード

- 各マスターの「新規登録 CSV テンプレート」を取得
- ヘッダー行のみで、データ行は空
- Shin さんが Excel で記入 → CSV インポートで一括投入する流れ

#### 用途 ②: 現在のデータの一括エクスポート

- 各マスターの全データ（または絞り込んだデータ）を CSV で出力
- バックアップ・編集・外部システム連携に活用
- Excel で一括編集 → CSV インポートで上書きする流れ

### 3.2 CSV インポート

- ファイルアップロード（最大サイズ要検討）
- バリデーション（必須列、データ型、外部キー整合性）
- ドライラン機能（インポート前に成功 / 失敗を表示）
- エラー行のスキップ + 詳細ログ出力
- 進捗バー表示（既存 `CsvImportJob` の `processedRows` / `totalRows` を活用）
- 完了通知

### 3.3 ジョブ管理

既存スキーマを活用:

- `CsvImportJob` モデル
  - ジョブの実行履歴
  - 成功 / 失敗 / 進行中ステータス
  - エラー詳細（JSON）
- `DataExportJob` モデル
  - エクスポート履歴
  - 生成ファイルの保存先（Cloudflare R2）
  - ファイル有効期限

## 4. UI 設計の方向性

### 4.1 マスター詳細画面に追加するボタン

各マスター一覧画面のヘッダーに以下を追加:

```
[新規作成] [テンプレート DL] [エクスポート] [インポート]
```

### 4.2 CSV インポート画面の流れ

```
ステップ 1: ファイル選択
  ↓
ステップ 2: ドライラン（プレビュー + バリデーション結果）
  ↓
ステップ 3: 実行確認
  ↓
ステップ 4: 進捗表示
  ↓
ステップ 5: 結果サマリー
```

### 4.3 ジョブ履歴画面

- 過去のインポート / エクスポート履歴を一覧表示
- 成功 / 失敗の結果確認
- エクスポートファイルの再ダウンロード（有効期限内）

## 5. 技術的検討事項

### 5.1 CSV ライブラリ

- `papaparse`: 既にプロジェクトに導入済み？要確認
- もしくは Node 標準の `csv-parse` / `csv-stringify`

### 5.2 ファイルストレージ

- エクスポート CSV: Cloudflare R2 の `shunya-prod-documents` バケット
- 一時保存期間: 7 日間（要検討）

### 5.3 大量データへの対応

- 1 ファイルあたりの行数上限: 10,000 行（暫定）
- バッチ処理（1,000 行ずつコミット）
- 非同期処理（バックグラウンドジョブ）

### 5.4 エンコーディング

- 入出力: UTF-8 BOM 付き（Excel 対応）
- 文字化け対策

### 5.5 文字種

- カラム名: 英語（システム内部の field 名と一致）
- データ: 多言語対応（UTF-8 で日本語・英語・中国語・ベトナム語）
